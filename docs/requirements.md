# 在庫管理アプリ「Zaico」要件定義書

## 1. 目的
シンプルかつ直感的に操作できる在庫管理システムを提供し、ユーザーが手軽に商品の在庫状況を把握・更新できるようにする。

## 2. ターゲットユーザー
- 小規模な店舗の店主
- 自宅の備品管理を行いたい個人ユーザー
- 複雑な在庫管理システムを必要としない現場担当者

## 3. 機能要件

### 3.1. ホーム画面（在庫一覧）
- 登録されている全商品の一覧表示
- 各商品の「商品名」と「現在の在庫数」を表示

### 3.2. 在庫操作機能
- **在庫数の増加**: 各商品行にある「＋」ボタンをワンクリックすることで、在庫数を1増やす。
- **在庫数の減少**: 各商品行にある「ー」ボタンをワンクリックすることで、在庫数を1減らす（在庫数は0未満にならない）。

### 3.3. 商品管理機能
- **商品の追加**:
    - 新しい商品名を入力して在庫リストに追加できる。
    - 初期在庫数を設定できる。
- **商品の削除**:
    - 不要になった商品をリストから削除できる。

## 4. 非機能要件
- **シンプルさ**: 説明書がなくても直感的に操作できるUI/UX。
- **レスポンス**: 在庫数の増減ボタン押下時に即座に画面に反映されること。

## 5. データモデル（案）
- **StockItem (在庫アイテム)**
    - ID (識別子)
    - Name (商品名)
    - Quantity (在庫数)
